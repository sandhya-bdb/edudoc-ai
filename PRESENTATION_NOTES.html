Core Technical Deep-Dive

A. The 3-Stage Classification Pipeline classifier.py

"This is the brain of the project. Instead of sending every document to an expensive LLM, we use a Cascade Architecture."
Key Points:
  • Stage 1: Rules Engine (Fastest) – Checks filenames for prefixes like cert_ or transcript_. Cost: $0, Latency: <1ms.
  • Stage 2: Local OCR (Moderate) – Uses EasyOCR to scan for educational keywords (e.g., "University", "Degree"). Cost: $0 (local), Latency: ~1-2s.
  • Stage 3: Generative AI (Powerhouse) – If the first two fail, we use Google Gemini Pro Vision to "look" at the document and understand its context. Cost: Low (pay-per-use), Latency: ~2-3s.

B. High-Concurrency API with Memory Safety api.py

"We built the backend with FastAPI and Asynchronous Python. We specifically optimized it for hardware with limited memory (like a MacBook Air)."
Key Points:
  • Semaphore Control (Line 124): We use asyncio.Semaphore to limit how many documents are processed at once. This prevents the server from crashing if a user uploads 100 images at once.
  • Streaming Responses (Line 156): Instead of making the user wait for all files to finish, we "stream" the results back to the browser one by one as they are ready.

C. Modern UI & User Experience index.html

"The frontend is a lightweight, responsive SPA (Single Page Application) that provides real-time feedback."
Key Points:
  • Glassmorphism Design: Uses modern CSS backdrops and gradients for a premium "Apple-like" feel.
  • NDJSON Handling: The Javascript handles the streaming backend response, updating the UI row-by-row as classification happens.
